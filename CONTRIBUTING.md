# Contributing to MCP-BRON Integration Spec

Thank you for your interest in contributing to the MCP-BRON integration specification!

## How to Contribute

### Reporting Issues

If you find gaps, inconsistencies, or unclear requirements in the specification:

1. Check existing issues to avoid duplicates
2. Open a new issue with a clear description
3. Include specific references to sections that need improvement
4. Suggest improvements when possible

### Proposing Enhancements

To propose enhancements to the specification:

1. Open an issue describing your proposed enhancement
2. Explain the use case and benefits
3. Wait for feedback before submitting a pull request

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-enhancement`)
3. Make your changes to the specification documents
4. Ensure your changes are clear and well-documented
5. Commit your changes (`git commit -m 'Add enhancement description'`)
6. Push to your branch (`git push origin feature/your-enhancement`)
7. Open a pull request with a clear description

### Contributing Workflows

We especially welcome contributions of new agentic workflows:

1. Design the workflow with clear objectives
2. Define input schemas and parameters
3. Document step-by-step BRON query sequences
4. Provide decision frameworks and risk assessment criteria
5. Include integration examples for AI agents
6. Add code samples demonstrating usage

### Sharing Implementation Experiences

If you've implemented the MCP-BRON gateway:

1. Share your experiences in discussions
2. Document integration patterns you've discovered
3. Report any challenges or deviations from the spec
4. Suggest improvements based on real-world usage

## Code of Conduct

Be respectful and constructive in all interactions. We're building this together.

## Questions?

Open an issue or start a discussion if you have questions about contributing.
